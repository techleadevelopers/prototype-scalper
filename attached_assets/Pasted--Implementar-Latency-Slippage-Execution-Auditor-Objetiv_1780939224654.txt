No mass sniper:
- se spread atual > spread médio aceitável, penalizar.

# Implementar Adaptive Symbol Rotation

Objetivo:
Fazer o sniper concentrar execução nos símbolos que estão pagando melhor agora, em vez de distribuir trades igualmente entre todos os pares.

Em scalp curto, nem todo símbolo está operável o tempo todo. O sistema deve rotacionar capital para símbolos com melhor momentum, melhor execução e melhor resultado recente.

## 1. Criar módulo `symbol_rotation.py`

Responsabilidade:
Calcular um ranking dinâmico dos símbolos permitidos.

Inputs:
- performance recente por símbolo/lado
- win rate recente
- profit factor recente
- PnL USDT recente
- score médio do Coach Ranker
- execução/slippage por símbolo
- spread médio
- volume/momentum atual
- taxa de sinais válidos
- drawdown recente
- toxicidade de contexto
- número de posições abertas

Retorno:

```json
{
  "activeSymbols": ["SOL-USDT", "ETH-USDT", "HYPE-USDT"],
  "reducedSymbols": ["BTC-USDT", "NEAR-USDT"],
  "pausedSymbols": ["MELANIA-USDT"],
  "ranking": [
    {
      "symbol": "SOL-USDT",
      "sideBias": "LONG",
      "rotationScore": 0.82,
      "allocationWeight": 0.24,
      "maxPositions": 4,
      "reason": "high_recent_pf_low_slippage"
    }
  ]
}
2. Criar rotationScore
rotationScore =
  recentPnlScore * 0.25
+ recentProfitFactorScore * 0.20
+ coachScoreAvg * 0.20
+ executionQualityScore * 0.15
+ currentMomentumScore * 0.10
+ liquidityScore * 0.10
- drawdownPenalty
- slippagePenalty
- toxicContextPenalty
3. Estados por símbolo
Cada símbolo deve ter estado:

HOT: aumentar prioridade e permitir mais posições
ACTIVE: operar normal
REDUCED: operar só setups fortes
PAUSED: não operar temporariamente
RECOVERY: operar pequeno até recuperar
Regras:

HOT: PF recente alto, execução boa, momentum bom
REDUCED: sinais medianos ou slippage elevado
PAUSED: sequência ruim, drawdown alto, execução ruim
RECOVERY: saiu de paused, volta com size menor
4. Integração com Mass Sniper
Antes de executar /sniper/mass:

consultar ranking de símbolos
aplicar allocationWeight
ordenar candidatos por aggressiveScore * rotationScore
limitar posições por símbolo conforme estado
Exemplo:

HOT: max 4-5 posições
ACTIVE: max 2-3 posições
REDUCED: max 1 posição
PAUSED: 0 posição
5. Side Bias
O sistema deve aprender lado por símbolo.

Exemplo:

SOL-USDT LONG performa bem no regime atual
SOL-USDT SHORT está ruim
então aumentar LONG e reduzir SHORT
Retornar:

{
  "symbol": "SOL-USDT",
  "longScore": 0.81,
  "shortScore": 0.42,
  "sideBias": "LONG"
}
6. Janela recente
Usar múltiplas janelas:

15 min: momentum atual
1h: execução recente
4h: performance tática
24h: estabilidade
Não deixar uma janela curta pausar símbolo bom por ruído.
