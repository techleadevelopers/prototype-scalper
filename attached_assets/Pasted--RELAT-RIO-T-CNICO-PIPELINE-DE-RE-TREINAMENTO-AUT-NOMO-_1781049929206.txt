O Motor de Otimização da Geometria do GatilhoO modelo (Random Forest / Gradient Boosting) usará as tags de desfecho para criar novas métricas de penalidade e recompensa na função de perda (loss function). O foco não é acertar a direção do mercado, mas sim ajustar o tamanho da armadilha.1. Lidando com EXPIRED_UNFILLED (Otimização de Fill Rate)O que significa: A IA previu a exaustão, jogou o gatilho muito longe e o mercado subiu sem espetar a ordem. O robô perdeu a oportunidade (opportunity cost).Ajuste Matemático: Se o padrão de candles de 1m/5m/15m gera repetidos cancelamentos por expiração, o modelo aprende a reduzir o multiplicador de distância do gatilho:$$\text{Distância Trigger} = \text{ATR}_{1m} \times (K - \epsilon)$$2. Lidando com FILLED_AND_STOPPED (Mitigação de Atropelamento)O que significa: O preço espetou o gatilho, mas tinha tanta força invisível no fluxo (sentimento agressivo) que rasgou o Stop Loss de 20% da margem.Ajuste Matemático: O modelo mapeia a densidade de ticks e o tamanho das sombras dos candles anteriores. Ele aprende que, sob aquele nível de volatilidade, o gatilho precisa ser posicionado mais fundo na região de suporte, ou que a ordem deve expirar mais rápido (expirationSeconds reduzido de 20s para 12s) porque o movimento é um rompimento real, não uma exaustão.3. Lidando com FILLED_AND_WON (Reforço Estatístico)O que significa: Execução perfeita. O espeto pegou a ordem limite (taxa Maker) e a recuperação curta pagou os $0.50 cents (10% na margem).Ajuste Matemático: O modelo eleva o peso das features multiframe que causaram essa vitória, aumentando o confidence e a kellyFraction na próxima vez que esse cenário exato se repetir no .env de símbolos.🛠️ III. import joblib
from datetime import datetime, timedelta
from sklearn.ensemble import RandomForestClassifier
from my_db_module import DatabaseClient

class QuantBrainAutonomicLearner:
    def __init__(self, model_path="artifacts/shadow_model.pkl"):
        self.model_path = model_path
        self.db = DatabaseClient()

    def perform_daily_recalibration(self):
        print(f"[🧠 LEARNER] Iniciando ciclo de re-treinamento autônomo: {datetime.now()}")
        
        # 1. Coleta apenas os sinais resolvidos nas últimas 24h/7 dias
        raw_data = self.db.query_resolved_signals(since=datetime.now() - timedelta(days=7))
        
        if len(raw_data) < 100:
            print("[⚠️ LEARNER] Amostras insuficientes para re-treino seguro. Abortando.")
            return

        X = [] # Matriz de Features Multiframe (1m, 5m, 15m, Ticks, Sentiment)
        y = [] # Target transformado: 0 = Falha/Stop/Expirado, 1 = Sucesso Puro (WON)

        for record in raw_data:
            features = record["multiframe_features"]
            desfecho = record["outcome"] # FILLED_AND_WON, FILLED_AND_STOPPED, EXPIRED_UNFILLED
            
            X.append(features)
            # O modelo aprende estritamente a caçar o lucro líquido positivo
            y.append(1 if desfecho == "FILLED_AND_WON" else 0)

        # 2. Treinamento incremental / Substituição controlada
        new_model = RandomForestClassifier(n_estimators=150, max_depth=8, random_state=42)
        new_model.fit(X, y)
        
        # 3. Guardrail de Sanidade: Só substitui o modelo em produção se a acurácia geométrica melhorar
        if self.validate_model_safety(new_model, X, y):
            joblib.dump(new_model, self.model_path)
            print("[✅ LEARNER] Novo cérebro calibrado e salvo em produção com sucesso!")
