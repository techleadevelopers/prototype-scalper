Objetivo:
Criar um motor de experimentos para testar variações do sniper em Demo VST com controle estatístico. O sistema deve descobrir quais configurações realmente melhoram PnL, win rate, profit factor, MFE/MAE e execução.

Não queremos mudar config no escuro. Queremos rodar experimentos controlados.

## 1. Criar `experiment_engine.py`

Responsabilidade:
Distribuir sinais/trades entre políticas diferentes e comparar resultados.

Exemplos de políticas:
- TP curto vs TP maior
- SL fixo vs SL adaptativo
- stacking depth 1/2/3/5
- aggressiveScore threshold 0.52/0.58/0.65
- trailing on/off
- symbol rotation on/off
- exit intelligence on/off
- market order vs filtered market order
- BTC regime strict vs loose

## 2. Assignment determinístico

Cada `signalId` ou `campaignId` deve cair sempre no mesmo grupo:

```txt
hash(signalId) % 100
0-49 = control
50-99 = treatment
Evitar viés de seleção.
