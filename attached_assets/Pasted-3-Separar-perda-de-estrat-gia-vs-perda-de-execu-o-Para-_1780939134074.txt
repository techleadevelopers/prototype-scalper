3. Separar perda de estratégia vs perda de execução
Para cada trade fechado, gerar:

{
  "strategyWouldHaveWon": true,
  "executionCausedLoss": false,
  "entrySlippageBps": 4.2,
  "exitSlippageBps": 3.1,
  "latencyDamagePct": 0.08,
  "executionQuality": "ACCEPTABLE_EXECUTION"
}
Regras:

Se MFE após sinal atingiu TP, mas entrada real foi atrasada e trade perdeu: execução matou.
Se nunca houve MFE relevante: entrada/setup ruim.
Se bateu TP teórico mas monitor não fechou: saída/monitor ruim.
Se slippage + fee consumiram edge: custo matou.

## 7. Aplicar ao sniper em massa

No mass sniper:
- se spread atual > spread médio aceitável, penalizar.
- se símbolo tem histórico de slippage alto, reduzir prioridade.
- se latência atual está alta, reduzir quantidade por ciclo.
- se priceMoveDuringLatency estimado for alto, evitar entrada atrasada.
- se `executionProfitDragPct` está alto, aumentar minScore do símbolo.
