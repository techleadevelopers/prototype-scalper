Depois: Só rejeita quando ambas condições são verdadeiras: separação pequena e score absoluto fraco (sideScore < candleMinScore). Score absoluto forte anula o chop check.

5 Gargalos Resolvidos — Resumo Técnico
Fix 1 — Coach Ranker: Morte por Inércia eliminada (coach_ranker.py)
Antes: penalidade cheia disparada em n=50 amostras, suficiente para enterrar ativos promissores com sequência ruim inicial.

Depois (3 mudanças simultâneas):

Rampa de confiança a(n) = min(1, (n-50)/100) — em n=50, penalidade é zero; em n=150, penalidade plena
Bayesian cold-start floor — learning score mistura 25% do prior neutro (0.50) em n=0, decaindo para 0% em n=50, evitando crash por ruído inicial
Threshold tóxico elevado de 30→50 amostras para declarar contexto tóxico com penalidade cheia
Fix 2 — signal_learning.py: Targets ATR-Normalizados (signal_learning.py)
Antes: TARGETS_USDT = (0.5, 1.0, 2.0) — BTC com $100k e TRUMP com $10 avaliados pelo mesmo alvo absurdo em USDT.

Depois: _atr_normalized_targets() gera targets como [0.30×ATR, 0.60×ATR, 1.00×ATR] convertidos para USDT via notional. BTC (ATR 0.8%, notional $100) → [$0.24, $0.48, $0.80]. TRUMP (ATR 5%, notional $100) → [$1.50, $3.00, $5.00]. Fallback automático para TARGETS_USDT se ATR indisponível.

Fix 3 — knowledge_base.py: WAL Mode + Busy Timeout (knowledge_base.py)
Antes: SQLite com journal mode padrão (DELETE) → escrita trava banco inteiro, shadow_sampler vs Node.js = database is locked silencioso.

Depois: init_db() executa antes de qualquer DDL:

PRAGMA journal_mode=WAL;   -- leituras concorrentes livres
PRAGMA busy_timeout=5000;  -- 5s de retry antes de abortar
PRAGMA synchronous=NORMAL; -- durabilidade segura com WAL
PRAGMA cache_size=-32000;  -- 32MB cache por conexão

Fix 4 — edge_gate.py: Expiry Fast-Path (edge_gate.py)
Antes: signal_expired calculado na linha 764, mas judge_entry só chamado na linha 978 — 200+ linhas de feature extraction, evaluate_sniper_window, ML inference desperdiçadas.

Depois: early return imediato após linha 764 quando signal_expired=True:

if signal_expired and not intelligence_only:
    return {"decision": "WAIT", "blocks": [...], "mode": "expired", ...}

Corta centenas de ms de processamento + I/O de mercado antes mesmo de tocar o pipeline.

Fix 5 — score_calibration.py: Calibração Regime-Aware (score_calibration.py)
Antes: penalidades de bucket calculadas sem contexto de regime — consolidação recente matava entradas no início de bull/bear markets.

Depois: _detect_regime_shift() compara distribuição dos últimos 30 trades vs. histórico completo. Quando regime mudou (≥60% dos recentes no novo regime): transition_factor = 0.35 → penalidades reduzidas em 65%. calibrate_score() também reduz deságio ECE de 50%→20% em transição.
